# 社科文献库使用指南
## 结合 AI 工具用于日常科研的小白手册

> 适用人群：社会学、人口学及相关领域的研究生，有文献检索需求，不需要编程基础。

---

## 这个文献库能帮你做什么？

这个数据库收录了社会学和人口学领域 **25 本核心英文期刊、2000 年至今、共 34,000+ 篇文章**的元数据（标题、摘要、作者、DOI）。

**它能帮你：**
- 快速了解某个议题在这些期刊上发表了哪些文章
- 配合 AI 做文献综述、找研究空白、梳理理论脉络
- 用关键词在 3 万多篇文章里秒级检索

**它不能做的：**
- 不包含全文 PDF（只有元数据）
- 不涵盖所有社科期刊，只有列表中的 25 本
- 部分早期文章摘要可能为空

## 方式一：直接在网页上浏览（无需任何设置）

打开：**https://huwenbo-lab.github.io/publication/**

这是一个在线浏览页面。新版前端会优先使用浏览器内 SQLite 检索引擎，支持按主题、作者、期刊、年份筛选；如果当前部署没有发布 `literature.db`，页面会自动回退到浏览模式。

## 方式二：本地全文检索（推荐，需要一次性安装）

这种方式可以在 **3 万多篇文章的标题、摘要和作者字段中同时搜索**，支持多关键词组合、按期刊和年份过滤，毫秒级返回结果。

### 第一步：安装 Python

如果你的电脑还没有 Python，前往 https://www.python.org/downloads/ 下载并安装（选最新版本，一路点下一步即可）。

安装完成后，打开终端（Mac：在应用程序→实用工具中找到"终端"；Windows：搜索"命令提示符"或"PowerShell"），输入：

```
python --version
```

看到版本号（如 `Python 3.12.0`）就说明安装成功。

### 第二步：进入文献库文件夹

在终端中切换到本文件夹（把下面的路径替换成你实际存放的位置）：

```bash
cd /你的路径/期刊文献
```

### 第三步：构建检索索引（只需做一次）

```bash
python build_search_db.py
```

等待约 10 秒，看到 `✓ 已建索引：34,222 篇文章 → literature.db` 就完成了。这会在文件夹里生成一个 `literature.db` 文件，后续搜索都靠它。

### 第四步：开始搜索

```bash
# 搜索关键词（同时在标题、摘要和作者里搜索）
python build_search_db.py --search "education inequality China"

# 限制返回结果数量（默认返回 20 条）
python build_search_db.py --search "marriage fertility" --limit 10

# 只看某本期刊
python build_search_db.py --search "stratification" --journal "American Journal of Sociology"

# 限定年份范围
python build_search_db.py --search "labor market" --year-from 2015 --year-to 2023

# 组合使用
python build_search_db.py --search "gender wage gap" --journal "American Sociological Review" --year-from 2010
```

**搜索技巧：**
- 多个词默认是"都包含"关系：`education inequality` 找同时含这两个词的文章
- 搜索精确短语要加引号：`"social mobility"` 找这个完整短语
- 用 OR 扩大范围：`marriage OR cohabitation`
- 用 NOT 排除：`fertility NOT mortality`

## 方式三：把文献库喂给 AI，让 AI 帮你做文献综述

这是最强大的用法。下面介绍几种场景。

### 场景 A：用网页端 AI（如 Claude、ChatGPT）做某个议题的综述

**操作步骤：**

1. 先用本地检索找到相关文章：

   ```bash
   python build_search_db.py --search "intergenerational mobility China" --limit 30
   ```

2. 把搜索结果复制粘贴给 AI，然后提问：

   > 以下是我从社会学期刊数据库检索到的关于中国代际流动的文章列表（含标题、期刊、年份和摘要片段）。请帮我：
   > 1. 梳理这个议题的主要研究问题和理论框架
   > 2. 找出近五年的研究趋势
   > 3. 识别可能的研究空白
   >
   > [粘贴搜索结果]

3. 根据 AI 的回答，用 DOI 去 Google Scholar 或图书馆找全文。

---

### 场景 B：让 AI 从文献库文件中直接检索（适合 Claude.ai）

文献库在 GitHub 上提供了按期刊整理的文章文件，可以直接把链接发给支持网页读取的 AI。

**示例：想了解《Demography》近年的研究动态**

把以下链接告诉 AI：
```
https://raw.githubusercontent.com/huwenbo-lab/publication/main/lit_db/abstracts/2020_2026/Demography.md
```

然后提问：
> 这是 Demography 期刊 2020-2026 年发表文章的标题和摘要。请帮我总结这本期刊近年最关注哪些议题，并列出与"生育率下降"相关的文章。

**25 本期刊的文件链接格式：**
```
# 标题索引（所有年份）
https://raw.githubusercontent.com/huwenbo-lab/publication/main/lit_db/titles/by_journal/期刊名.md

# 摘要（2020-2026年）
https://raw.githubusercontent.com/huwenbo-lab/publication/main/lit_db/abstracts/2020_2026/期刊名.md

# 摘要（2010-2019年）
https://raw.githubusercontent.com/huwenbo-lab/publication/main/lit_db/abstracts/2010_2019/期刊名.md

# 摘要（2000-2009年）
https://raw.githubusercontent.com/huwenbo-lab/publication/main/lit_db/abstracts/2000_2009/期刊名.md
```

把"期刊名"替换成实际期刊名，例如：
- `American Sociological Review`
- `Journal of Marriage and Family`
- `Demography`

（期刊名列表见文末附录）

### 场景 C：研究设计阶段——用 AI 帮你找研究空白

**操作步骤：**

1. 确定你感兴趣的两三个关键词，分别搜索：

   ```bash
   python build_search_db.py --search "single mothers labor" --limit 20
   python build_search_db.py --search "lone parent employment" --limit 20
   ```

2. 把两次搜索结果都发给 AI，提问：

   > 我在研究单身母亲的劳动力市场参与问题。以下是我从社会学期刊数据库检索到的相关文献列表。请帮我：
   > 1. 现有研究主要关注了哪些国家和人群？哪些地区研究较少？
   > 2. 用了哪些主要理论框架？
   > 3. 在研究方法上有什么共同点或局限？
   > 4. 根据这些文献，你认为有哪些值得进一步研究的问题？

---

### 场景 D：写文献综述段落

1. 搜索核心关键词，保存结果：

   ```bash
   python build_search_db.py --search "assortative mating education" --year-from 2010 --limit 25
   ```

2. 复制给 AI，提问：

   > 以下是关于教育同质婚（assortative mating）的近年期刊文章。我正在写一篇关于中国婚配模式变迁的论文的文献综述部分。请帮我用学术语言写一段 300-400 字的英文文献综述，重点涵盖：（1）教育同质婚的整体趋势；（2）与不平等的关系；（3）中国的相关发现（如有）。

3. 检查 AI 的引用是否准确（AI 可能会根据摘要推断而非完整阅读原文），根据 DOI 去查原文核实关键论点。

---

### 场景 E：检查你的参考文献是否在数据库里

写论文时，可以用 DOI 或标题关键词快速确认某篇文章是否在收录范围内：

```bash
# 用标题关键词确认
python build_search_db.py --search "The Fading American Dream" --journal "American Sociological Review"
```

## 使用 AI 时的注意事项

**AI 可能出错的地方：**
- AI 根据摘要推测的论文观点，不一定等同于论文的真实结论，写论文时务必查阅全文
- 如果摘要字段为空（部分早期文章），AI 只能根据标题判断相关性
- 文献库收录的是元数据，不包含方法、数据、表格等正文内容

**好的提问方式：**
- 给 AI 明确的任务（综述、找空白、分类、对比）比让它"帮我找文献"更有效
- 告诉 AI 你的研究背景和目标受众（硕士论文、期刊投稿、综述文章等）
- 一次不要给 AI 太多文章（20-30 篇是比较合适的范围），质量比数量重要

## 更新文献库

文献库定期从 CrossRef API 获取最新文章。如果你想手动更新到最新：

```bash
# 进入文件夹
cd /你的路径/期刊文献

# 激活虚拟环境（如果有）
source venv/bin/activate   # Mac/Linux
venv\Scripts\activate      # Windows

# 抓取最近 30 天新文章
python update.py

# 更新后重建检索索引
python build_search_db.py --rebuild
```

## 遇到问题？

- **Python 找不到命令**：确认 Python 安装时勾选了"Add to PATH"选项；Mac 用户可能需要用 `python3` 代替 `python`
- **搜索结果为空**：尝试减少关键词数量，或换用更宽泛的词
- **literature.db 不存在**：先运行 `python build_search_db.py` 建立索引
- **摘要显示为空**：部分文章（尤其是 2010 年前）确实没有录入摘要，可直接用 DOI 去 Google Scholar 查看
